Port Registry & System Architecture

All containers, ports, domains, and file paths for the KYCPlus deployment on a single AWS EC2 instance.

Complete Port Registry
Container Name Host Port Domain Type Env Source Folder
kycplus_backend_qa 4000 qa.kycplus.in/api/ Node.js Backend QA /home/ubuntu/QA/kycplus_backend2
kycplus_backend 4001 demo.kycplus.in/api/ Node.js Backend Demo /home/ubuntu/Demo/kycplus_backend2
kycplus_frontend_qa 3000 qa.kycplus.in React/Vite Frontend QA /var/www/QA/kycplus_frontend
kycplus_frontend_demo 3002 demo.kycplus.in React/Vite Frontend Demo /var/www/Demo/kycplus_frontend
kyc-rekyc-frontend 8086 qarekyc.kycplus.in ReKYC Frontend QA /var/www/QA/kycplus_rekyc
kyc-rekyc-frontend-demo 8087 demorekyc.kycplus.in ReKYC Frontend Demo /var/www/Demo/kycplus_rekyc
face-ai-face-api-1 8000 *.kycplus.in/face/ Face AI (FastAPI/Python) Shared /home/ubuntu/face-ai
Nginx listens on ports 80 and 443. All above ports are internal — not exposed to internet directly.
Request Flow (How Traffic Reaches Containers)

demo.kycplus.in

Browser → HTTPS :443
  → Nginx (server_name: demo.kycplus.in)
    /api/*  → localhost:4001  (Demo Backend)
    /face/* → localhost:8000  (Face AI)
    /*      → localhost:3002  (Demo Frontend)

qa.kycplus.in

Browser → HTTPS :443
  → Nginx (server_name: qa.kycplus.in)
    /api/*  → localhost:4000  (QA Backend)
    /face/* → localhost:8000  (Face AI)
    /*      → localhost:3000  (QA Frontend)

demorekyc.kycplus.in

Browser → HTTPS :443
  → Nginx (server_name: demorekyc.kycplus.in)
    /api/*  → localhost:4001  (Demo Backend)
    /*      → localhost:8087  (Demo ReKYC)

qarekyc.kycplus.in

Browser → HTTPS :443
  → Nginx (server_name: qarekyc.kycplus.in)
    /api/*  → localhost:4000  (QA Backend)
    /*      → localhost:8086  (QA ReKYC)

Full Folder Structure on EC2
/home/ubuntu/
├── Demo/
│   └── kycplus_backend2/        # Demo backend — .env, docker-compose.yml, src
│       ├── .env                  # PORT=4001, DB_*, JWT_*, AWS_*
│       └── docker-compose.yml
├── QA/
│   └── kycplus_backend2/        # QA backend — .env, docker-compose.yml, src
│       ├── .env                  # PORT=4000
│       └── docker-compose.yml
└── face-ai/                     # Face recognition API (shared)
    └── docker-compose.yml

/var/www/
├── Demo/
│   ├── kycplus_frontend/        # Demo KYC frontend
│   │   ├── .env                  # VITE_API_URL, VITE_API, PORT=3002
│   │   └── docker-compose.yml
│   └── kycplus_rekyc/           # Demo ReKYC frontend
│       ├── .env                  # VITE_API, FRONTEND_PORT=8087
│       └── docker-compose.yml
└── QA/
    ├── kycplus_frontend/        # QA KYC frontend
    │   ├── .env                  # VITE_API_URL, VITE_API, PORT=3000
    │   └── docker-compose.yml
    └── kycplus_rekyc/           # QA ReKYC frontend
        ├── .env
        └── docker-compose.yml

/etc/nginx/
├── sites-available/             # config files (one per domain)
└── sites-enabled/               # symlinks to active configs
.env Variable Reference

Backend .env (applies at runtime)

PORT=4001
NODE_ENV=production
DB_HOST=host.docker.internal
DB_PORT=3306
DB_NAME=kycdb
DB_USER=kycuser
DB_PASS=password
JWT_SECRET=secret
AWS_REGION=ap-south-1
Change → restart container only. No rebuild.

Frontend .env (bakes into bundle at build)

VITE_API_URL=https://demo.kycplus.in/api
VITE_API=https://demo.kycplus.in/api
VITE_ASSET_URL=https://demo.kycplus.in
VITE_ENV=PROD
VITE_FACE_API_URL=https://dev.kycplus.in/face
VITE_CLIENT_SESSION_STORAGE_KEY=session_data
VITE_CLIENT_AUTH_STORAGE_KEY=accessToken
PORT=3002
Change → must rebuild Docker image.

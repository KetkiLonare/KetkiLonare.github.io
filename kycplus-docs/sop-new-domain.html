✅ Verify
Before you start — verify these
1
DNS A record points to this server

Go to your domain registrar / Route53. Add an A record: newapp.kycplus.in → server-public-ip. Wait for propagation (usually 1–5 min for Route53).

# Verify DNS resolves to correct IP
nslookup newapp.kycplus.in
# OR
dig newapp.kycplus.in +short
2
Pick ports not already in use

Each app on the server needs unique host ports. Check what's taken:

Current KYCPlus ports taken: 3000, 3002, 4000, 4001, 8000, 8086, 8087. Pick something else.
3
Ensure Docker and Nginx are installed

If missing: apt install docker.io nginx -y

4
Code is in Git — pull it to server
Pre-deployment checklist
  • DNS A record created and verified
  • Backend port 4002 is free on this server
  • Frontend port 3003 is free on this server
  • Code pulled from Git to server
  • Dockerfile exists in backend and frontend repos
  • docker-compose.yml exists in backend and frontend repos
  • .env.example or list of required variables is available
  • AWS Security Group allows ports 80 and 443 inbound

Docker & Server Concepts for Beginners

🐳 What is Docker?

Docker packages your app + all its dependencies into a container — a sealed, isolated environment. It runs the same way on any server. You don't install Node.js or Python directly on the server — Docker handles everything inside the container.

Analogy

Docker container = shipping container. App inside is isolated. The EC2 server is the ship carrying multiple containers.

📦 Image vs Container

Image = the blueprint (built from Dockerfile). Container = the running instance of that image. You build an image once, run it as a container. To update code: rebuild the image, restart the container.

📄 docker-compose.yml

A config file that defines how to run a container — which port to expose, which .env to use, container name, restart policy. Run docker compose up -d in the same folder to start everything defined in it.

🔌 Ports — HOST:CONTAINER

Format: HOST_PORT:CONTAINER_PORT. Example: 4001:4001 means traffic hitting the server's port 4001 goes into the container's port 4001. Nginx listens on 80/443 and routes to internal container ports.

Analogy

Ports = office building doors. Nginx is the receptionist at the front door (80/443) who directs visitors to the right room (container port).

🌐 Nginx Reverse Proxy

Nginx sits in front of all containers. When someone visits demo.kycplus.in, nginx receives it on port 443 (HTTPS) and forwards to the right container: frontend on 3002, backend on 4001, face API on 8000.

⚠️ Why Frontend Needs Rebuild After .env Change

Backend reads .env at runtime (when container starts) → restart is enough.
Frontend (React/Vite) bakes VITE_ variables into the JavaScript bundle at BUILD time. Changing .env after build has zero effect. You MUST rebuild the Docker image.

❌ 502 Bad Gateway

Container not running OR nginx proxy_pass port is wrong.

❌ API URL wrong / missing /api

VITE_API or VITE_API_URL in frontend .env missing /api suffix, OR was changed after build without rebuild.

Fix: edit .env, ensure value ends with /api, then rebuild the frontend image.
